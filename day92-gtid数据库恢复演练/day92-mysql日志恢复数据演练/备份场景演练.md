# 基于mysqldump全量+binlog增量实现数据备份恢复

```
核心理念就是

1. 每天12点做好全量备份

2. 第二天的数据变动，写入binlog，全部基于GTID记录信息

3. 如果发生了删除数据行为，剔除那一次的GTID版本号即可。

4. 具体如更复杂的场景，如删库后，又继续发生了数据变动，这个留作以后深入DBA方向再思考

5.目前能实现

- 全量数据恢复
- 增量数据恢复
- 剔除误删除数据的操作即可
```
